# Experiments

Small, focused code experiments — quick ways to test a concept without building a full project.

## How to Use

Each experiment is a self-contained notebook or script in its own subfolder:

```
experiments/
├── exp-001-embeddings-comparison/
│   ├── notebook.ipynb
│   └── README.md
├── exp-002-chunking-strategies/
│   └── ...
```

## Naming Convention

`exp-{number}-{descriptive-name}/`

Examples:
- `exp-001-embeddings-comparison`
- `exp-002-chunking-strategies`
- `exp-003-prompt-chaining`
